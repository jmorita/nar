## 試算（モデル評価）標準手順

モデリングノートブックで試算セルを実装する際は、以下の4ブロックを順番に出力すること。

### 用語定義
- `pred_prob`: モデルの予測確率（単勝的中確率）
- `pred_odds`: モデルの予測オッズ
- `recommend_score`: `pred_odds * pred_prob`（期待値指標、1.0超で理論上プラス）
- `actual_hit`: 実際の的中フラグ（0/1）
- `actual_odds`: 実際の払戻オッズ（的中時のみ有効）
- 回収額: `actual_odds * actual_hit` の合計（母数100円賭けベース）
- 回収率: `回収額 / 母数` × 100（%）
- 評価: 区分の pred_prob 上下限に対して実際的中率を比較
  - 実際的中率 > 区分上限 → **過小評価**（青）
  - 実際的中率 < 区分下限 → **過大評価**（橙）
  - 区分内 → **適正**（緑）

---

### ブロック1: pred_prob 10パーセンタイル区分分析

`pd.qcut(pred_prob, 10)` で10等分し、区分ごとに以下を表形式で出力。

| 列 | 内容 |
|----|------|
| 区分下限 | パーセンタイル区間の下限 |
| 区分上限 | パーセンタイル区間の上限 |
| 母数 | レコード数 |
| 的中数 | actual_hit の合計 |
| 的中率 | 的中数 / 母数（%） |
| 回収額 | 合計回収額 |
| 回収率 | 回収率（%） |
| 評価 | 過大評価 / 過小評価 / 適正 |

**色分け**:
- 回収率: 120%超→濃緑, 100-120%→薄緑, 80-100%→黄, 80%未満→赤
- 評価: 過小評価→青背景, 過大評価→橙背景, 適正→緑背景

列: 区分下限, 区分上限, 母数, 的中数, 的中率, 回収額, 回収率

**色分け**: 回収率のみ（ブロック1と同基準）

### ブロック3: pred_prob × recommend_score 5×5クロス分析（全データ）

**テーブル出力**（各セル: 母数 / 的中率 / 回収率 / 評価）:
- 行: pred_prob 区分（低→高）

- 母数, 的中率, 回収率, 評価（過大:-1 / 適正:0 / 過小:+1）を個別にヒートマップ化
- 各サブプロットにカラーバーと軸ラベル（区分の中央値）を付与

## 仮説検証 ベースアプローチ（個別変数 × 結果指標）

新規特徴量の有効性確認、ドメイン仮説の裏付け、モデル改善方針決定のための **標準テンプレ**。
リファレンス実装: mra プロジェクトの `#M701_hypothesis_verification_v01.ipynb` の `hypothesis_analyze()` 関数（JRA 用にデータ結合部分のみ差し替えて流用）。

### 区分の作り方（変数タイプ自動判定）

| 変数タイプ | 区分方法 |
|---|---|
| 連続変数（数値かつ `nunique > 20`） | `pd.qcut(..., q=10, duplicates='drop')` で **10パーセンタイル区分**。各区分の **下限・上限** を列として出力 |
| カテゴリ変数（object またはユニーク値 ≤ 20） | 各コード値ごとに集計。コード値数が **20 超** の場合は上位 20 のみ表示、残りは **「その他」** に集約 |

### 出力指標（各区分／コード値ごと）

| 列 | 内容 |
|---|---|
| 区分下限／区分上限 | 連続変数のみ（パーセンタイル区間境界） |
| 母数 | 該当サンプル数 |
| 1着回数 | `is_top1 == 1` のレコード数 |
| 1着率 (%) | 1着回数 / 母数 |
| 2着以内率 (%) | `is_top1 or is_top2` の割合 |
| 3着以内率 (%) | `is_top1 or is_top2 or is_top3` の割合 |
| 単勝回収率 (%) | `mean(odds × is_top1) × 100`（100円賭けベース） |
| 平均配当 (円) | `is_top1=1` のみの `odds × 100` の平均 |

### 色分けルール（pandas Styler）

**1着率と単勝回収率は必ず色分けすること**（仮説検証時の必須要件）。

**配色方針**: ダーク／ライト両テーマで視認できるよう、背景色 + 文字色を **両方明示**する。濃色背景には `color: #ffffff`（白）、それ以外は `color: #000000`（黒）を必ず指定。

- **1着率**（最重要・絶対値スケール）:
  - `>= 30%`: 濃緑 bg=`#2e7d32` fg=`#ffffff`
  - `>= 20%`: 緑   bg=`#66bb6a` fg=`#000000`
  - `>= 15%`: 薄緑 bg=`#a5d6a7` fg=`#000000`
  - `>= 10%`: 黄   bg=`#fff176` fg=`#000000`
  - `< 10%` : 赤   bg=`#ef9a9a` fg=`#000000`
- **単勝回収率**（最重要・絶対値スケール）:
  - `>= 120%`: 濃緑 bg=`#2e7d32` fg=`#ffffff`
  - `>= 100%`: 緑   bg=`#66bb6a` fg=`#000000`
  - `>=  80%`: 薄緑 bg=`#a5d6a7` fg=`#000000`
  - `>=  60%`: 黄   bg=`#fff176` fg=`#000000`
  - `<  60%` : 赤   bg=`#e57373` fg=`#000000`
- **2着以内率／3着以内率／平均配当**: 全体平均との比で色分け
  - `>= 1.15`: 緑   bg=`#66bb6a` fg=`#000000`
  - `>= 1.05`: 薄緑 bg=`#a5d6a7` fg=`#000000`
  - `>= 0.95`: 中庸（無色）
  - `>= 0.85`: 黄   bg=`#fff176` fg=`#000000`
  - `< 0.85` : 赤   bg=`#ef9a9a` fg=`#000000`

### 適用ガイドライン

- 新仮説は **1 行で検証**：`hypothesis_analyze(df, "<feat>")` のみ。連続／カテゴリ判定は自動。
- データセット: JRA は出走表特徴量 + 結果（着順・払戻オッズ）を `race_id × umaban` で結合した `df`。`is_top1/2/3` は確定着順から導出、`odds_num` は単勝確定オッズ。
- 層別分析: `df_subset = df[df["umaban"]==1]` のように絞ってから関数を呼ぶ。
- 結果の解釈: 「区分上限の `pred_prob` 等よりも実際 1着率が高い → 過小評価／低い → 過大評価」の対称ルールで判定。
